# 工具使用指南

## 概述

Paper2Skill 工具是一个专业的论文/专利信息抽取系统，旨在帮助用户从科研文献和专利中快速提取结构化的技能知识。本指南将概述工具的使用方法和工作流程，帮助您快速上手。

## 工具组成

Paper2Skill 工具由以下核心组件组成：

1. **PDF 转换模块**：负责将 PDF 文档转换为可编辑的文本格式
2. **HTML 处理模块**：处理本地保存的 HTML 网页文件
3. **LLM 处理模块**：利用大语言模型分析和处理文档内容
4. **Skill 提取模块**：从处理后的文本中提取结构化的技能信息
5. **文档生成模块**：生成规范化的 Markdown 文档

## 工作流程

### 1. 准备阶段

- **安装依赖**：确保所有必要的软件和库都已正确安装
- **准备文档**：收集需要处理的 PDF 格式论文/专利或 HTML 格式网页文件
- **配置环境**：按照环境设置指南配置工具运行环境

### 2. 处理阶段

1. **PDF 转换**（适用于 PDF 文档）：
   - 运行 PDF 转换命令
   - 检查转换结果，确保文本质量

2. **HTML 处理**（适用于网页文件）：
   - 准备 HTML 文件，按照平台/账号/文章的目录结构组织
   - 运行 HTML 处理命令
   - 检查处理结果，确保内容完整

3. **LLM 处理**：
   - 选择合适的模型参数
   - 运行 LLM 处理命令
   - 监控处理过程，确保正常完成

4. **Skill 提取**：
   - 配置提取参数
   - 运行 Skill 提取命令
   - 检查提取结果，进行必要的调整

### 3. 结果阶段

- **文档查看**：打开生成的 Markdown 文档
- **内容调整**：根据需要修改文档内容
- **技能应用**：将提取的技能知识应用到实际场景

## 目录结构

项目使用以下目录结构组织文件：

```
├── html/                  # HTML 网页文件目录
│   └── platform/          # 按平台分类（如微信、小红书等）
│       └── account/       # 按账号分类（如公众号名称）
│           └── article/    # 文章目录
│               └── article.html  # 保存的 HTML 网页文件
├── llm_processed/        # LLM 处理后的文档
│   └── publisher/         # 按出版社/平台分类
│       └── journal/       # 按期刊/账号分类
│           └── paper/     # 论文/文章目录
│               ├── paper.md     # 处理后的 Markdown 文档
│               └── chapters/    # 章节文件
└── paper_skills/          # 提取的技能文档
    ├── by_topic/          # 按主题分类
    ├── by_method/         # 按方法分类
    └── by_journal/        # 按期刊/账号分类
```

## 注意事项

- **处理时间**：LLM 处理可能需要较长时间，取决于文档大小和系统性能
- **模型选择**：根据文档类型和内容选择合适的模型参数
- **结果调整**：自动提取的结果可能需要人工调整，以确保准确性
- **资源需求**：处理大型文档可能需要较多的内存和计算资源

## 常见问题

### 1. PDF 转换失败
- **原因**：PDF 加密或格式异常
- **解决**：尝试使用不同的转换参数，或手动处理特殊格式

### 2. LLM 处理超时
- **原因**：文档过大或系统资源不足
- **解决**：分割文档为较小部分，或增加系统资源

### 3. Skill 提取不完整
- **原因**：文档结构复杂或模型参数不合适
- **解决**：调整提取参数，或对文档结构进行预处理
